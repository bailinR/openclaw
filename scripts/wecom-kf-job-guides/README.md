# 企业微信招聘提示词与岗位文档

bridge 启动时读取本目录的提示词和岗位文档。修改后需要重启 bridge 才会生效。

## 文件分工

- `_global.md`：全局提示词。放角色身份、面试语气、回复风格、固定开场流程。
- `_job-overview.md`：当前所有岗位概述。放岗位名称、岗位别名、对应岗位文档和流程用途。
- `sales.md`、`project-planning.md` 等：具体岗位文档。放该岗位的流程安排、题目、分值、评分标准和结果判定。

## 运行时组装顺序

1. 先读取 `_global.md`，获得全局规则和固定开场话术。
2. 再读取 `_job-overview.md`，判断候选人应聘哪个岗位。
3. 匹配到岗位后读取对应岗位文档，例如 `sales.md`。
4. 前台回复只生成求职者可见消息；资料沉淀和评分由后台异步任务处理。

## 新增岗位

1. 在 `_job-overview.md` 增加岗位名称、别名和对应文件。
2. 新建一个岗位 `.md` 文件。
3. 在岗位文件中写清楚岗位别名、流程安排、题目、分值、评分标准。
4. 重启 bridge。
